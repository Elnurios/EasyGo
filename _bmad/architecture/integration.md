# Архитектура интеграции EasyGo + TaxiMaster

## Концепция запуска

На старте проекта EasyGo арендует готовое программное решение **TaxiMaster**
для операционной деятельности такси. Собственная разработка фокусируется на
**личном кабинете водителя** — ключевом дифференциаторе платформы.

---

## Двухуровневая архитектура

```text
УРОВЕНЬ 1 — TaxiMaster (арендованное ПО, быстрый запуск)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Приложение водителя    →  выполнение и получение заказов
  Приложение пассажира   →  заказ такси
  Диспетчерская панель   →  управление заказами
  База данных            →  водители, пассажиры, поездки, маршруты

УРОВЕНЬ 2 — Разработка EasyGo (с первого дня)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Личный кабинет водителя  →  аналитика, кооперативный функционал
  EasyGo Backend           →  агрегация данных, расчёт прибыли
  API-интеграция           →  получение данных из TaxiMaster
```

## Поток данных

```text
TaxiMaster API
      │
      │  polling каждые 30–60 секунд
      │  (webhooks отсутствуют — используем периодические запросы)
      ▼
EasyGo Backend (собственная БД)
      │  - собирает и агрегирует поездки
      │  - рассчитывает себестоимость по профилю водителя
      │  - рассчитывает чистую прибыль
      │  - хранит кооперативные данные (доля, голоса, рефералы)
      ▼
Личный кабинет водителя EasyGo
      - финансовая аналитика
      - операционная статистика
      - кооперативный блок
      - аналитика рынка
```

## Данные из TaxiMaster API

### Аутентификация

- Метод: MD5-подпись параметров запроса + секретный ключ
- Заголовок: `Signature`
- Опционально: `X-User-Id` для разграничения доступа
- Формат ответа: JSON `{ "code": 0, "descr": "...", "data": {} }`
- Минимальная версия TaxiMaster: 3.15

---

### Доступные данные для личного кабинета водителя

| Блок кабинета | Данные | TaxiMaster Endpoint |
| ------------- | ------ | ------------------- |
| Профиль водителя | ФИО, телефон, документы, статус | `get_driver_info` |
| Финансы | Баланс, счета, KPI | `get_driver_info` → `accounts`, `kpi` |
| Текущие заказы | Активные поездки в реальном времени | `get_current_orders` |
| История поездок | Выполненные заказы, маршруты, стоимость | `get_completed_orders` |
| Расчёт стоимости | Цена поездки по маршруту | `calc_order_cost2` |
| Автомобиль | Данные авто, уровень топлива | `get_car_info` → `fuel_level` |
| GPS / Местоположение | Текущие координаты экипажа | `get_crew_coordinates` |
| Смены | Управление рабочими сменами | `create_shift` |

---

### Что разрабатываем сами (нет в TaxiMaster API)

| Функция | Решение |
| ------- | ------- |
| Агрегированная статистика (сумма за период, кол-во поездок) | Собираем через `get_completed_orders`, считаем в EasyGo Backend |
| Себестоимость поездки (12 ₽/км или 17 ₽/км) | Рассчитываем в EasyGo Backend по профилю водителя |
| Чистая прибыль водителя | Выручка из TaxiMaster минус себестоимость EasyGo |
| Real-time обновления | Polling каждые 30–60 сек (webhooks в TaxiMaster отсутствуют) |
| Рейтинг и отзывы пассажиров | Собственная система EasyGo |
| Кооперативный блок (доля, голосования, рефералы) | Полностью собственная разработка |
| Аналитика рынка (мониторинг цен, рекомендации) | Полностью собственная разработка |

## Блоки личного кабинета водителя

### Блок 1 — Финансовая аналитика

| Показатель | Источник |
| ---------- | -------- |
| Выручка сегодня / неделя / месяц (грязная) | TaxiMaster → `get_completed_orders` |
| Себестоимость поездок | EasyGo: кол-во км × 12 или 17 ₽/км |
| **Чистая прибыль** | Выручка − себестоимость (расчёт EasyGo) |
| Средний доход в час vs целевой 700 ₽/час | Расчёт EasyGo |
| Реферальные начисления (2%) | EasyGo Backend |
| Доля прибыли кооператива | EasyGo Backend |

### Блок 2 — Операционная статистика

| Показатель | Источник |
| ---------- | -------- |
| Количество поездок за день / месяц | TaxiMaster → `get_completed_orders` |
| Пробег рабочий / холостой | TaxiMaster → данные заказов |
| Средняя длина поездки (км, мин) | TaxiMaster → данные заказов |
| Время онлайн / офлайн | TaxiMaster → смены (`create_shift`) |
| KPI водителя | TaxiMaster → `get_driver_info` → `kpi` |

### Блок 3 — Расходы и себестоимость

| Показатель | Источник |
| ---------- | -------- |
| Профиль водителя (своё авто / аренда) | EasyGo (выбирается при регистрации) |
| Ставка себестоимости | 12 ₽/км — своё авто, 17 ₽/км — аренда |
| Уровень топлива в автомобиле | TaxiMaster → `get_car_info` → `fuel_level` |
| Ручной ввод расходов (ТО, штрафы, мойка) | EasyGo — форма ввода в кабинете |

### Блок 4 — Кооперативный блок

| Показатель | Источник |
| ---------- | -------- |
| Статус пайщика и дата вступления | EasyGo Backend |
| Открытые голосования Совета водителей | EasyGo Backend |
| Начисленная доля прибыли платформы | EasyGo Backend |
| Реферальная статистика (кто приглашён, начисления) | EasyGo Backend |

### Блок 5 — Аналитика рынка (от EasyGo)

| Показатель | Источник |
| ---------- | -------- |
| Рекомендуемые зоны спроса | EasyGo Backend (анализ заказов) |
| Рекомендуемое время работы | EasyGo Backend |
| Мониторинг рыночных цен на маршрутах | EasyGo Backend |

## Этапы разработки

### Этап 1 — Быстрый запуск (TaxiMaster)

- Настройка TaxiMaster под концепцию EasyGo (0% комиссия в тарифах)
- Подключение API-ключей и настройка аутентификации
- Запуск приложений водителя и пассажира

### Этап 2 — Личный кабинет водителя (параллельно)

- Разработка EasyGo Backend: сервис polling + агрегации данных из TaxiMaster
- Разработка UI личного кабинета: финансовый блок + операционная статистика
- Расчёт себестоимости и чистой прибыли по профилю водителя

### Этап 3 — Кооперативный функционал

- Добавление кооперативного блока в кабинет (доля, голосования)
- Реферальная система с начислением 2%
- Распределение прибыли платформы

### Этап 4 — Аналитика и экосистема

- Мониторинг рыночных цен
- Рекомендации по зонам и времени работы
- Интеграция экосистемы услуг (топливо, ТО, страховка)
